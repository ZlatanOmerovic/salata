# Playground — Salata

## Overview

A one-command Docker-based interactive environment for trying Salata without any local installation. Builds an Ubuntu container with Rust, all supported runtimes, popular editors, and a pre-built salata binary. Drops the user into bash in a ready-to-use workspace.

## Directory Structure

```
playground/
  ├── Dockerfile.playground             # Full build image
  ├── docker-compose.playground.yml     # Single interactive service
  ├── start-playground.sh               # Linux/macOS launcher
  ├── start-playground.bat              # Windows CMD launcher
  ├── start-playground.ps1              # PowerShell launcher
  └── README.md                         # User-facing playground docs
```

## Container Contents

### Base
- Ubuntu latest

### Build Tools
- curl, git, build-essential, pkg-config, libssl-dev
- Latest stable Rust via rustup

### Runtimes (all latest available via apt/nodesource/rustup)

| Runtime | Package | Binary |
|---------|---------|--------|
| Python | python3 + pip | `python3` |
| Ruby | ruby | `ruby` |
| Node.js | nodesource LTS | `node` |
| TypeScript | npm: typescript, ts-node, tsx | `ts-node`, `tsx` |
| PHP | php + php-cgi | `php`, `php-cgi` |
| Shell | bash, dash, zsh, fish | `/bin/bash`, etc. |

### Editors (no custom configs, default setups only)
- nano
- vim / vi
- neovim (nvim)
- emacs-nox (terminal only, no GUI)

### Shell Prompt & CLI Tools
- [Starship](https://starship.rs/) — cross-shell prompt, installed via `curl -sS https://starship.rs/install.sh | sh -s -- -y`
- Initialized in `/root/.bashrc` with `eval "$(starship init bash)"`
- Default config only (no custom starship.toml) — shows git branch, language versions, command duration, exit status out of the box
- Nerd Font not required — Starship falls back gracefully to ASCII symbols
- [bat](https://github.com/sharkdp/bat) — cat alternative with syntax highlighting
- Installed via `apt install -y bat`, symlinked: `ln -s /usr/bin/batcat /usr/local/bin/bat`
- No custom config, default setup only

### Salata
- Full source tree copied to `/opt/salata`
- Built with `cargo build --release` inside the container
- Release binaries (`salata`, `salata-cgi`, `salata-fastcgi`, `salata-server`) added to PATH
- `salata init` run in `/home/playground` to generate starter config + files

### Helper Scripts
- `rebuild-salata` — recompiles from `/opt/salata` source and installs all 4 binaries to PATH

## Working Directory

`/home/playground` — user lands here on login. Contains:
- `config.toml` — generated by `salata init` with all runtimes enabled
- `index.slt` — starter file generated by `salata init`
- `errors/` — default 404.slt and 500.slt
- `examples/` — comprehensive CLI and web examples (copied from source tree)
- `workspace/` — volume-mounted for file persistence between sessions

## Shell Configuration

The container sets `HOME=/home/playground`. Bash customizations (Starship init, welcome banner) go in `/home/playground/.bashrc`. A `/home/playground/.bash_profile` sources `.bashrc` for login shells (`bash --login`).

## docker-compose.playground.yml

```yaml
services:
  playground:
    build:
      context: ..
      dockerfile: playground/Dockerfile.playground
    stdin_open: true
    tty: true
    ports:
      - "3000:3000"
    volumes:
      - ./workspace:/home/playground/workspace
```

- Interactive mode (stdin_open + tty) for bash session
- Port 3000 exposed for `salata-server` testing
- `workspace/` volume for file persistence

## Launcher Scripts

### start-playground.sh (Linux/macOS)
```bash
#!/usr/bin/env bash
set -e

if ! command -v docker &>/dev/null; then
    echo "Error: Docker is not installed or not in PATH."
    exit 1
fi

echo "Starting Salata Playground..."
docker compose -f playground/docker-compose.playground.yml run --rm --service-ports playground bash

echo "Salata Playground exited."
```

### start-playground.bat (Windows CMD)
Same logic: check for docker, run compose with interactive bash, print exit message.

### start-playground.ps1 (PowerShell)
Same logic: check for docker, run compose with interactive bash, print exit message.

All launcher scripts must be executable and handle missing Docker gracefully.

## Testing the Playground

After build, verify:
1. Container starts and Starship prompt appears (not default root@... prompt)
2. Welcome banner prints with detected runtime versions
3. `salata index.slt` produces output
4. All 6 runtimes work (python3, ruby, node, ts-node, php, bash)
5. All 4 editors are installed (nano, vim, nvim, emacs)
6. `bat index.slt` shows syntax-highlighted output
7. `salata-server . --port 3000` starts and serves on localhost:3000
8. Files created in `workspace/` persist after container restart
9. `salata init --path test-project` creates a new project directory
